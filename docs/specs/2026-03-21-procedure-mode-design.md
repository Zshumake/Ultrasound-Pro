# Procedure Mode — Bedside Quick Reference

## Problem

Residents use the app in two contexts: studying at home (deep learning) and at the bedside (quick reference during live procedures). The current detail page is optimized for studying — long, content-rich, scroll-heavy. At the bedside, residents need specific info in under 3 seconds and can't scroll through a full teaching page.

The #1 thing residents look for mid-procedure: **how to find the correct ultrasound view** (landmarking steps).

## Solution

A **Procedure Mode** toggle on each detail page that collapses the full teaching content into a single-screen, bedside-optimized card.

## Procedure Mode Card — Content & Order

When toggled on, the page shows exactly 7 blocks following the actual procedure workflow:

### 1. SUPPLIES
- Interactive checklist with checkboxes (same as study mode)
- Residents tick items as they gather them onto the tray

### 2. PATIENT POSITIONING
- **Image slot** with camera icon placeholder
- **Detailed photo description**: exact patient position, what landmarks should be visible in the frame, orientation notes
- Generated from existing `positioning` field, expanded into explicit photography instructions
- Example: *"Patient supine, head of bed 30 deg, arm relaxed at side. Frame: shoulder and upper arm visible, forearm supported on pillow."*

### 3. PROBE PLACEMENT
- **Image slot** with camera icon placeholder
- **Detailed photo description**: probe type, exact placement on body, orientation relative to landmarks, hand grip position
- Generated from existing `probe` + `landmarking` fields
- Example: *"Linear probe (6-13 MHz) placed transversely just anterior to lateral acromial edge. Frame: show probe orientation relative to acromion, include hand grip."*

### 4. FIND THE VIEW
- Numbered landmarking steps (concise)
- This is the most critical section — the thing residents need most during a live procedure

### 5. CORRECT US IMAGE
- **Image slot** with camera icon placeholder
- **Detailed photo description**: what the ultrasound screen should show, key structures to label, orientation markers
- Generated from existing `correctImage` field
- Example: *"Acromion at screen edge, supraspinatus fibers parallel and uniform. Label: bursal stripe between deltoid (above) and supraspinatus (below)."*

### 6. NEEDLE CORRIDOR
- Where to aim, entry point, trajectory, target

### 7. AVOID
- Red-highlighted block, impossible to miss
- Critical structures to stay away from

**Everything else is hidden**: tips, pearls, video, 3D model, anatomy diagram, US gallery, tags, indications. These are study-mode content.

## UI Design

### Toggle
- Floating action button (FAB) or app bar toggle button
- Icon: stethoscope or bolt/lightning for "quick mode"
- Label: "PROCEDURE MODE" in monospaced text
- Persists user preference via SharedPreferences (if they toggle on, it stays on next time)

### Layout
- Title + category color bar at top (compact)
- 7 blocks stack vertically with tight padding
- Target: fit on a single phone screen without scrolling (or minimal scroll)
- Smaller font sizes than study mode, no decorative elements
- Dark mode only assumption (procedure rooms are dim) — but respect theme setting

### Image Slots
- Prominent bordered box with dashed border and camera icon
- Photo description text below the box in a muted but readable style
- When actual images are added later, the description becomes a caption/overlay
- These descriptions serve dual purpose: (1) tells the resident what they should see, (2) serves as a shot list for capturing clinical photos

### AVOID Block
- Red background tint, red border, warning icon
- Same styling as study mode but more compact

## Data Model

No new fields needed. All content comes from existing fields:
- `supplies` -> Supplies checklist
- `positioning` -> Patient positioning description + photo instructions
- `probe` -> Probe placement description + photo instructions
- `landmarking` -> Find the View steps
- `correctImage` -> US image description + photo instructions
- `corridor` -> Needle corridor
- `avoid` -> Avoid list

Photo descriptions will be generated by combining existing field content with standardized framing/photography instruction templates.

## Implementation Notes

- New widget: `ProcedureModeView` — takes an `InjectionTechnique`, renders the compact card
- Toggle state: `bool _isProcedureMode` in `InjectionDetailPage` state, persisted via SharedPreferences
- The toggle swaps between the current `CustomScrollView` content and the `ProcedureModeView`
- Animation: quick crossfade between modes
- Keyboard shortcut: `P` key toggles procedure mode (consistent with existing keyboard shortcuts)

## Success Criteria

- A resident in a procedure room can toggle on Procedure Mode and find the landmarking steps within 3 seconds
- The card fits on a standard phone screen (375px wide) without horizontal scroll
- Image slot descriptions are specific enough to serve as a photography shot list
- Toggle preference persists across sessions
